### Code Quality Rules (CRITICAL)
1. Every page must be a Server Component by default (`"use client"` only when needed for state/effects)
2. Every page MUST export `metadata` with `title` and `description`
3. Use `PageHeader` for page titles — don't hand-roll
4. Use `Card`, `StatCard`, `ChartCard` for layouts — no raw divs with shadow classes
5. Use real SVG icons only — no emojis. (Helper: `@/components/Layouts/sidebar/icons` exports many, including `HomeIcon`, `User`, `Calendar`, `Table`, `PieChart`, `FourCircle`, `BriefcaseIcon`, `ShoppingBagIcon`, `UsersIcon`, `WalletIcon`, `CloudIcon`, `MegaphoneIcon`, `CpuIcon`, `LifeBuoyIcon`, `TruckIcon`, `MailIcon`, `ChatIcon`, `KanbanIcon`, `StickyNoteIcon`, `CheckSquareIcon`, `FolderIcon`, `ContactBookIcon`, `TicketIcon`, `FileTextIcon`, `PackageIcon`, `ShoppingCartIcon`, `StarIcon`, `TagIcon`, `BoxIcon`, `ShieldIcon`, `CreditCardIcon`, `ActivityIcon`, `BellIcon`, `PlugIcon`, `InputIcon`, `ToggleIcon`, `CalendarPlusIcon`, `UploadIcon`, `EditIcon`, `CheckIcon`, `LayersIcon`, `CandlestickIcon`, `GaugeIcon`, `RadioIcon`, `FileBarIcon`, `BtnIcon`, `BadgeIcon`, `CardIcon`, `AlertIcon`, `AvatarIcon`, `DropdownIcon`, `ModalIcon`, `DrawerIcon`, `TabsIcon`, `AccordionIcon`, `TooltipIcon`, `PopoverIcon`, `PaginationIcon`, `ProgressIcon`, `SpinnerIcon`, `ToastIcon`, `TimelineIcon`, `BannerIcon`, `CommandIcon`, `GlobeIcon`, `RocketIcon`, `BookIcon`, `HelpCircleIcon`, `PhoneIcon`, `WrenchIcon`, `MapPinIcon`, `LockIcon`, `LogInIcon`, `UserPlusIcon`, `KeyIcon`, `RefreshCwIcon`, `AlertTriangleIcon`, `XCircleIcon`, `ServerIcon`)
6. Light AND dark mode must work — use Tailwind `dark:` variants
7. Pages must be responsive — use `grid-cols-1 md:grid-cols-2 lg:grid-cols-3` etc.
8. Charts use `<HeliosChart options={...} series={...} type="..." height={...} />` — DO NOT use raw `Chart` from react-apexcharts
9. NO `console.log`, NO `any` types in shared code
10. Each page should have ~80-150 lines of meaningful content (not a stub)

- DO NOT create new design tokens — use what's in style.css
